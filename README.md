## 📖 О проекте

**Zabbix Desktop Monitor** — это нативное десктопное приложение, разработанное как удобная альтернатива стандартному веб-интерфейсу Zabbix. Позволяет в режиме реального времени контролировать состояние сетевой инфраструктуры прямо с рабочего стола — без браузера.

Проект выполнен в рамках производственной практики **ПМ.02 «Осуществление интеграции программных модулей»**.

> **Разработчик:** Ронжин М.А. — UI/UX, макет в Figma, UML-моделирование, Frontend-разработка (PyQt6)

## ✨ Возможности

### 🔐 Авторизация
- Безопасное подключение к Zabbix API с проверкой учётных данных
- Кнопка «Подключиться» блокируется при пустых полях
- Центрированный экран входа в современном тёмном стиле

### 📋 Мониторинг узлов
- Таблица всех хостов с колонками: **Имя / IP / Группа / Состояние**
- Цветовая индикация статуса:
  - 🟢 **Доступен** — агент/ICMP отвечает, нет активных триггеров
  - 🔴 **Проблема** — сработал триггер Zabbix
  - ⚫ **Выключен** — хост отключён в мониторинге
- Фильтрация по имени / IP (с задержкой debounce — без лагов)
- Фильтрация по группе через выпадающий список

### 📊 Графики метрик
- **Адаптивный набор метрик** — автоматически определяет тип хоста:
  - Хосты с агентом: `CPU` / `RAM` / `Диск` / `Сеть`
  - ICMP-хосты (без агента): `ICMP ping` / `ICMP response time` / `ICMP loss` / `PING response time`
- Показывает **только те графики, для которых есть реальные данные** (пустые слоты скрываются)
- Выбор интервала: 1 ч / 3 ч / 6 ч / 12 ч / 24 ч
- Все графики загружаются параллельно в фоновых потоках — UI не зависает

### 📥 Экспорт данных
- Выгрузка в **CSV** (разделитель `;`, кодировка UTF-8 BOM — открывается в Excel)
- Выгрузка в **Excel (.xlsx)** с цветным оформлением строк по статусу
- Экспортируется текущий отфильтрованный список (не все хосты)
- При ошибке прав доступа — понятное сообщение пользователю

### 💬 Комментарии к узлам
- Сохранение текстовых заметок к любому хосту
- Данные хранятся в локальной базе SQLite (`comments.db`)
- Комментарии сохраняются между перезапусками

### ⏱️ Автообновление
- Настраиваемый интервал (от 5 сек до 1 часа)
- Защита от параллельных запросов: если обновление ещё идёт — новый тик пропускается

### ⌨️ Горячие клавиши
| Комбинация | Действие |
|:---:|:---|
| `Ctrl+F` | Фокус на поле поиска |
| `Ctrl+E` | Экспорт в CSV |
| `Ctrl+R` | Обновить список узлов |

### 📝 Логирование
- Ошибки записываются в `app_errors.log`
- Ротация файла при достижении **5 МБ** (хранится 1 резервная копия)

## 🏗️ Архитектура

```
DesktopZabbix/
├── app.py               # GUI (PyQt6): окна, виджеты, потоки UI
├── zabbix_backend.py    # DAL: Zabbix API, SQLite, логика метрик
├── get_hosts.py         # Вспомогательный модуль хостов
├── get_metrics.py       # Вспомогательный модуль метрик
├── test_zabbix.py       # Тесты подключения
├── comments.db          # SQLite БД комментариев (создаётся автоматически)
└── app_errors.log       # Лог ошибок (создаётся автоматически)
```

Проект следует паттерну **MVC**:
- `zabbix_backend.py` — **Model** (данные, API, БД)
- `app.py` (классы окон) — **View** (интерфейс)
- `app.py` (класс `Controller`) — **Controller** (логика переключения экранов)

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/yokous/DesktopZabbix.git
cd DesktopZabbix
```

### 2. Создание виртуального окружения

**Linux / macOS:**
```bash
python3 -m venv venv
source venv/bin/activate
```

**Windows (PowerShell):**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

### 3. Установка зависимостей

```bash
pip install PyQt6 matplotlib pyzabbix openpyxl
```

### 4. Запуск

```bash
python app.py
```
